## 2. 이론별 평가

### 2.1 Global Workspace Theory (Baars/Dehaene)

**이론 요구사항**:
- 다수의 전문 처리기(모듈)가 동시에 활동
- "글로벌 워크스페이스"에서 정보가 전역적으로 방송(broadcast)
- 의식적 접근 = 정보가 워크스페이스에 진입하여 모든 모듈에 공유되는 순간
- 주의(attention)가 워크스페이스 진입을 게이팅

**현재 상태**:

| 요소 | 상태 | 평가 |
|------|------|------|
| 전문 처리기(모듈) | 구조적으로 존재 (V1, A1, Hippo, BG...) | 부분 충족 |
| 글로벌 워크스페이스 | 시상(Thalamus)이 릴레이 역할하지만 **방송 메커니즘 없음** | 미충족 |
| 전역 방송 | 정보가 단방향 전파만 함. 역방향/전역 공유 없음 | **미충족** |
| 주의 게이팅 | PPC_Attention 있지만 정적(학습 없음) + 비활성 | 미충족 |
| 점화(Ignition) | 임계점 넘으면 전역 활성화되는 메커니즘 없음 | **미충족** |

**갭**: 정보가 모듈 간 흐르지만, "방송"이 없음. 시각 정보가 해마까지 가지만,
해마의 기억이 시각 피질로 돌아와 지각을 변경하는 top-down 루프가 학습하지 않음.
워크스페이스에 해당하는 PFC/WM은 비활성(Phase 18 disabled).

**점수: 1/10**

---

### 2.2 Integrated Information Theory (Tononi)

**이론 요구사항**:
- 높은 통합 정보량 (Φ > 0)
- 시스템이 분할 불가능해야 함 (부분의 합 > 전체가 아닌, 전체 > 부분의 합)
- 인과적 구조: 각 상태가 과거에 의해 결정되고 미래를 결정

**현재 상태**:

| 요소 | 상태 | 평가 |
|------|------|------|
| 통합 | 모듈 간 연결 존재하지만 대부분 단방향 feedforward | 낮음 |
| 분할 불가능성 | Phase 15-20 제거해도 행동 변화 없음 → 분할 가능 | **매우 낮음** |
| 인과적 구조 | 98.2% 정적 → 과거 경험이 현재 상태에 영향 미미 | **매우 낮음** |
| 되먹임(recurrence) | 일부 recurrent 연결 있지만 정적 | 낮음 |

**갭**: IIT 관점에서 현재 뇌의 Φ는 거의 0에 가까움.
이유: 대부분의 연결이 정적이므로, 동일한 감각 입력에 항상 동일한 출력.
경험에 의한 상태 변화(=학습)가 거의 없으므로 인과적 구조가 빈약.

뇌를 반으로 나눠도(Phase 15-20 제거) 행동이 동일 → 통합이 아닌 병렬.

**점수: 0.5/10**

### 2.3 Predictive Processing (Friston/Clark)

**이론 요구사항**:
- 뇌는 감각 입력을 **예측**하는 생성 모델
- 예측 오차(prediction error)가 위로 전파되어 모델 업데이트
- 행동은 예측 오차를 최소화하기 위한 적극적 추론(active inference)
- 계층적: 하위 수준은 빠르고 구체적, 상위 수준은 느리고 추상적

**현재 상태**:

| 요소 | 상태 | 평가 |
|------|------|------|
| 감각 예측 | Self_Predict 존재하지만 비활성 + 학습 없음 | **미충족** |
| 예측 오차 | 계산하지 않음. 어디에서도 predicted - actual 연산 없음 | **미충족** |
| 오차 기반 학습 | R-STDP는 보상 기반이지 예측 오차 기반이 아님 | 미충족 |
| 계층적 예측 | V1→V2→V4→IT 계층 있지만 모두 feedforward + 정적 | 미충족 |
| 적극적 추론 | 행동이 예측 오차 최소화 목적이 아님 | 미충족 |

**갭**: 예측 처리의 핵심 요소가 하나도 구현되지 않음.
Self_Predict(Phase 20)가 이름만 "예측"이지, 실제로는 정적 시냅스를 통해
Motor 복사를 받을 뿐. 예측 오차를 계산하지도, 그것으로 학습하지도 않음.

이것이 가장 심각한 갭일 수 있음. 예측 처리는 의식뿐 아니라
**기본적인 지각과 학습의 메커니즘**이기도 하므로.

**점수: 0/10**

### 2.4 Higher-Order Theory (Rosenthal)

**이론 요구사항**:
- 의식 = 자신의 정신 상태에 대한 **표상의 표상**
- 1차 표상: "빨간 사과를 본다" (감각)
- 2차 표상: "나는 빨간 사과를 보고 있다는 것을 안다" (메타인지)
- 재귀적 자기 참조가 가능해야 함

**현재 상태**:

| 요소 | 상태 | 평가 |
|------|------|------|
| 1차 표상 | 감각 뉴런이 환경 정보 인코딩 (V1, A1 등) | 존재 |
| 2차 표상 | Meta_Confidence, Self_Narrative 존재하지만 비활성 + 정적 | **형식적** |
| 재귀적 참조 | Self_Narrative recurrent 있지만 자기 자신만 참조 | 미충족 |
| 학습하는 메타인지 | Hebbian 1개 (valence→confidence) — 비활성 | **미충족** |

**갭**: "메타인지"라는 이름의 뉴런 집단이 있지만, 그것이 실제로 자신의
정신 상태를 표상하지 않음. Meta_Confidence는 Assoc_Valence로부터 입력을 받아
활성화될 뿐, "내가 확신하고 있다"를 표상하는 것이 아님.

진정한 Higher-Order 표상은: 1차 처리 결과를 입력으로 받아 → 그것에 대한
판단을 생성하고 → 그 판단이 다시 1차 처리에 영향을 미치는 루프.
현재는 이 루프의 어떤 부분도 학습하지 않음.

**점수: 0.5/10 (구조만 존재)**

## 3. 핵심 질문: 학습 없는 고차 회로는 무엇인가?

### "학습하지 않는 메타인지는 메타인지인가?"

**아니다.** 메타인지의 정의는 "자기 인지 과정에 대한 인지"이다.
고정된 가중치로 특정 입력 조합에 반응하는 것은 **조건 반사**이지 메타인지가 아니다.

진정한 메타인지는:
- 경험을 통해 "나는 이 상황에서 틀리기 쉽다"를 **학습**
- 새로운 상황에서 자신의 불확실성을 **평가**
- 그 평가에 기반하여 전략을 **수정**

현재 Meta_Confidence는 valence 입력이 높으면 활성화되도록 **설계**되었을 뿐,
"높은 valence = 확신"이라는 관계를 **학습**한 것이 아님.

### "학습하지 않는 자기모델은 자기모델인가?"

**아니다.** 자기모델은 경험을 통해 "내가 어떤 존재인지"를 구성하는 것.
Self_Body가 에너지/배고픔 신호를 받아 활성화되는 것은 **센서**이지 자기모델이 아님.

자기모델은:
- "배고플 때 나는 위험을 무릅쓴다"를 경험에서 **학습**
- "나는 왼쪽으로 도는 경향이 있다"를 **관찰에서 발견**
- 이 지식을 미래 행동에 **적용**

### "학습하지 않는 Theory of Mind는 ToM인가?"

**아니다.** ToM은 타인의 정신 상태를 **추론**하는 것.
ToM_Intention이 NPC 근접도를 기반으로 활성화되는 것은 **근접 감지**이지 ToM이 아님.

## 4. Option A vs Option B 평가

### Option A: 기존 구조 유지 + 가소성 추가

**장점**:
- 599개 시냅스의 구조가 이미 존재 — 일부를 STDP로 전환하면 됨
- Phase 15-20 회로가 (비활성이지만) 뇌 영역 간 연결 토폴로지를 정의
- 점진적 접근 가능 — 하나씩 활성화하며 검증

**단점**:
- 수동 튜닝된 가중치와 STDP가 충돌 (가중치가 학습으로 이동하면 균형 붕괴)
- 588개 정적 시냅스를 하나씩 전환하는 것은 비현실적
- **근본 문제**: 구조 자체가 "학습을 위해 설계된 것이 아님". 학습 가능 토폴로지와 하드코딩 토폴로지는 다름
- Phase 12-14 재발 위험 (새 학습 시냅스가 Motor 간섭)

### Option B: 구조 축소 + 학습 극대화

**장점**:
- 처음부터 학습 중심으로 설계 — "능력은 학습에서 창발"
- 뉴런 수 감소 = 연산 빠름 = 더 많은 에피소드 훈련 가능
- Phase 15-20의 비활성 코드 제거 → 코드 복잡도 감소
- 프로젝트 철학("인간 뇌처럼 학습")에 부합

**단점**:
- Phase 1-14의 생존 회로까지 건드리면 기존 성과 손실
- "어디까지 줄이고 어디부터 학습을 켤 것인가" 판단이 어려움
- 급진적 변경 → 코드 안정성 위험

### 평가

**Option B가 의식에 더 가까움.** 이유:

1. 의식의 모든 이론이 **학습/적응**을 요구함. 정적 구조는 의식의 기반이 될 수 없음.
2. Option A는 "고정된 뼈대 위에 근육 붙이기". 뼈대가 학습용으로 설계되지 않았으므로 근육이 제대로 작동하지 않을 위험.
3. Option B는 "뼈대부터 학습에 맞게 재설계". 더 어렵지만 올바른 방향.

### 권장: 하이브리드 접근

- **Phase 1-14**: 유지 (생존 회로 + 기본 감각). 이 부분은 "척수/뇌간" 수준으로 하드코딩이 생물학적으로 정당화됨
- **Phase 15-20**: 코드 제거 또는 근본적 재설계 (학습 중심)
- **Phase L5+**: 핵심 피질 경로에 STDP 도입 (V1→IT, PFC→BG)
- **환경**: 고차 인지를 요구하는 환경 압력 동시 도입

## 5. 환경 복잡도 평가

### 현재 환경: ForagerGym

- 2D 400×400 맵
- 음식 5개 (랜덤 위치)
- 원형 통증 구역 2개
- NPC 에이전트 (Phase 15 대상, 현재 비활성)
- 관찰 공간: 왼쪽/오른쪽 음식, 벽, 통증 감지

### 이 환경에서 의식이 필요한가?

**아니다.** 이 환경은 다음으로 최적 해결 가능:
- 음식 방향으로 회전 (반사)
- 통증 반대 방향으로 회전 (반사)
- 벽 감지 시 회전 (반사)

→ 3개 반사 규칙이면 충분. 해마, 시각 피질, 언어, 메타인지가 전혀 불필요.

### 의식이 유용한 환경이란?

- **시간적 의존성**: "10스텝 전에 본 것을 기억해야 보상 획득" → WM 필수
- **범주 학습**: "빨간 음식은 독, 파란 음식은 보상" → 시각 학습 필수
- **사회적 추론**: "NPC가 음식을 숨기면 다른 곳을 찾아야" → ToM 필수
- **순서 과제**: "A → B → C 순서로 방문해야 보상" → 계획 필수
- **예측 가능성**: "음식이 매 50스텝마다 위치 변경" → 예측 모델 필수

**현재 환경에서는 어떤 고차 인지도 적응적 이점을 제공하지 않음.**
환경이 뇌를 만드는 것이지, 뇌가 환경을 만드는 것이 아님.

## 6. 결론 및 추천 방향

### 의식까지의 거리

| 이론 | 점수 | 가장 큰 갭 |
|------|------|-----------|
| Global Workspace | 1/10 | 전역 방송 메커니즘 없음 |
| IIT | 0.5/10 | 통합 정보 거의 0 (분할 가능) |
| Predictive Processing | 0/10 | 예측 오차 학습 완전 부재 |
| Higher-Order | 0.5/10 | 학습하는 메타 표상 없음 |

### 우선순위

1. **광범위한 시냅스 가소성** (모든 이론의 전제조건)
2. **예측-오차 학습 루프** (Predictive Processing — 가장 실용적)
3. **환경 복잡화** (학습이 유용한 상황 생성)
4. **전역 방송** (Global Workspace — 가소성 + 예측 위에 구축)

### 구체적 다음 단계

**Phase L5: 피질 STDP + 예측 오차**
- V1→V2→V4→IT 경로에 STDP 도입 (지각 학습)
- 예측 오차 계산 → 학습 신호로 사용
- Phase 15-20 비활성 코드 정리 (제거 또는 학습 중심 재설계)

**환경 업그레이드**
- 다양한 음식 종류 (학습이 필요한 구별)
- 시간적 패턴 (예측이 유용한 규칙성)
- 순서 과제 (계획이 필요한 복잡도)
